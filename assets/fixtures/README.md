# Fixtures

Pre-recorded sponsor responses, seed data, and sample prescription PDFs
the demo replays when `DEMO_FIXTURE_MODE=true`. Use these for:

- Offline dry-runs (coffee shop, plane, conference room with no WiFi)
- Pitch dry-runs (deterministic timing for the storyteller)
- Conference-WiFi fallback if a sponsor goes red at T-30

## Parser-format gap

The existing extractors in `apps/agent/src/tools/execute.py` look for
`Drug:`, `NDC:`, `Dose:`, and `ICD-10:` prefixes — which match the
`rx-lisinopril` / `rx-metformin` PDFs but **NOT** the more realistic
`rx-ozempic-martinez` / `rx-humira-thompson` ones (those use medical-letter
formatting with `PRESCRIBED MEDICATION` headers, `Sig:` directives, and
ICD-10 codes in tables, not labeled fields).

Dropping the realistic PDFs in fixture mode currently falls back to
`_stub_for_size` — which keys on file-size parity to return either
Lisinopril or Metformin. Fine for fixture mode (the demo always
produces a valid receipt), but misleading if you intend the audit page
to reflect the dropped chart.

**To fix when you wire up the realistic PDFs:** add letterhead-aware
extractors to `_local_parse` and `_sandbox_parse` (or have the planner
LLM extract the fields in live mode).

## Regenerating the PDFs

```bash
# Simple Lisinopril / Metformin pair (matches the current regex extractors)
python scripts/gen_demo_pdfs.py

# Realistic letterhead pair (Humira + insurance cards) — see parser gap above
python scripts/gen_mock_pdfs.py
```

Both write into `assets/fixtures/`.

## Synthetic-data discipline

- Patient names are always obviously synthetic (Jane Doe, John Sample,
  Mary Roe, Sarah Martinez, Lawrence Thompson).
- DOBs and member IDs are sequential demo values, never real.
- The dropzone helper text on the home page calls this out before the
  reviewer drops anything.
